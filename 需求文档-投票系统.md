## 2. 功能需求

### 2.1 管理员后台

#### 2.1.1 账号管理
- 管理员账号登录/退出
- 账号密码认证
- Session 会话管理

#### 2.1.2 创建投票
- 支持创建 **单选投票** 和 **多选投票**
- 投票内容：
  - 投票标题（必填，最多 100 字）
  - 投票说明（选填，最多 500 字）
  - 投票选项（最少 2 个选项，最多 20 个选项，每个选项最多 50 字）
  - 投票截止时间（选填，不设置则不限制）
  - **投票限制**：
    - 每位微信用户投票次数限制（创建时设置，如：每人1票、每人2票、每人无限票）
    - 基于微信 UnionID 识别用户
- 每个投票生成唯一的 **分享链接** 和 **二维码**

#### 2.1.3 投票列表
- 展示所有投票：标题、状态（进行中/已截止）、参与人数、创建时间
- 支持按状态筛选（全部/进行中/已截止）
- 支持搜索投票标题
- 可对投票进行：编辑、删除、查看明细操作

#### 2.1.4 投票结果与明细
- **结果概览**：总投票人数、各选项投票数量和占比、趋势图表
- **明细记录**：每个投票人的信息（微信昵称、头像、投票时间、选择的选项）
- 支持 **导出明细** 为 Excel

### 2.2 微信分享与投票（前端）

#### 2.2.1 分享功能
- 每个投票生成独立的 H5 页面链接
- 支持生成分享 **二维码**，可扫码直接投票
- 页面支持微信内打开（适配微信浏览器）
- 微信分享卡片显示自定义标题、描述和封面图

#### 2.2.2 投票页面
- **微信授权登录**：用户需微信授权获取 UnionID 才能投票
- 展示投票标题、说明、选项
- 投票规则提示（单选/多选、剩余投票次数）
- 提交后显示"投票成功"提示
- 已用完投票次数的用户不可再投票

#### 2.2.3 微信 JS-SDK 配置
- 微信分享参数签名（config、ready、error 处理）
- 分享到朋友、分享到朋友圈支持

## 3. 技术方案

### 3.1 技术栈
- **后端**：Node.js + Express + MySQL
- **前端**：React + Ant Design
- **微信集成**：微信 JS-SDK + 微信授权登录（OAuth2.0）
- **部署**：阿里云服务器（Nginx 反向代理）
- **二维码**：qrcode 库生成

### 3.2 数据库设计

#### 管理员表 (admins)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键自增 |
| username | VARCHAR(50) | 管理员用户名 |
| password | VARCHAR(255) | 密码（加密存储） |
| created_at | DATETIME | 创建时间 |

#### 投票表 (votes)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键自增 |
| title | VARCHAR(100) | 投票标题 |
| description | TEXT | 投票说明 |
| type | ENUM('single', 'multiple') | 单选/多选 |
| options | JSON | 选项列表 |
| max_votes_per_user | INT | 每位用户最大投票次数（0=无限） |
| end_time | DATETIME | 截止时间（NULL表示不限制） |
| status | ENUM('active', 'closed') | 状态 |
| share_title | VARCHAR(100) | 分享标题 |
| share_desc | VARCHAR(200) | 分享描述 |
| share_img | VARCHAR(255) | 分享封面图 |
| share_url | VARCHAR(255) | 分享链接 |
| qrcode | VARCHAR(255) | 二维码路径 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

#### 投票记录表 (vote_records)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键自增 |
| vote_id | INT | 关联投票ID |
| unionid | VARCHAR(100) | 微信用户唯一标识 |
| nickname | VARCHAR(100) | 微信昵称 |
| avatar | VARCHAR(255) | 微信头像 |
| options | JSON | 选择的选项 |
| created_at | DATETIME | 投票时间 |

## 4. 微信用户投票限制说明

- 用户身份识别：通过微信授权获取 UnionID
- 投票次数：在创建投票时设置（每位用户可投票次数：1-N 或无限）
- 限制逻辑：用户投票时查询该用户在该投票下的已投票次数，判断是否可投票

## 5. 非功能需求

- 页面加载时间 < 3秒
- 支持 1000+ 并发投票
- 数据安全：投票记录不可篡改，管理员密码加密存储
- 微信分享卡片显示自定义标题和封面图
